# API Documentation

## Overview

Create comprehensive README documentation with usage examples, API reference, and installation instructions to help developers understand and use the Hamming distance library effectively.

## Implementation Requirements

The README must include clear installation instructions, usage examples for both functions, complete API documentation with parameter descriptions and return values, error handling examples, and practical use cases for Hamming distance calculations.

## Acceptance Criteria

- README includes npm install instructions
- Usage examples show both hammingDistance and hammingDistanceBits functions
- API reference documents function signatures, parameters, and return types
- Error handling examples demonstrate proper exception catching
- Real-world use cases explain when to use each function
- Code examples are executable and correct
- Documentation follows standard JavaScript library conventions
- Examples include both simple and complex scenarios

## Testing Strategy

Documentation examples should be tested to ensure they execute correctly. Consider using documentation testing tools to validate that README code examples remain accurate as the library evolves.

## Web Interface Integration

The web interface should include embedded documentation, interactive examples that users can modify and run, and links to the full API documentation for deeper reference.
