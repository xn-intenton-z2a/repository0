# Node.js Modules & Package Management
## https://nodejs.org/api/esm.html
Comprehensive guide to ECMAScript modules (ESM) in Node.js, including `import`/`export` syntax, JSON modules via import assertions, conditional exports and the `exports` field in `package.json` to manage entry points for ESM and CommonJS. Essential for structuring library entry points, ensuring cross-environment compatibility, and customizing module resolution strategies in CLI and library code. Last updated Node.js v20. Authoritative Node.js documentation.

# Node.js Process API
## https://nodejs.org/api/process.html#processargv
Authoritative reference for accessing `process.argv`, `process.env`, signal events, and exit codes. Guides best practices for CLI argument parsing, environment variable management, error handling patterns, and process lifecycle controls. Critical for robust CLI utilities and automation scripts. Last reviewed Node.js v20.
## License: CC BY-SA 4.0

# CLI Argument Parsing Libraries
## https://github.com/substack/minimist, https://yargs.js.org/docs/, https://github.com/tj/commander
Comparative overview of leading Node.js CLI frameworks: lightweight `minimist` for quick flag parsing; `yargs` for command hierarchy, middleware, and autogenerated help; `commander` for nested commands and custom hooks. Demonstrates actionable patterns for aliasing, defaults, validation, and user-friendly output. Helps choose and implement the right parser for simple to complex CLIs. Last updated 2022–2024.
## License: MIT

# Configuration & Validation Libraries
## https://github.com/motdotla/dotenv, https://github.com/colinhacks/zod
Covers loading environment variables from `.env` with `dotenv`, addressing multi-environment setups, variable expansion, and security considerations, plus runtime schema validation with `zod` (numeric ranges, environment type parsing, error formatting). Essential for reliable configuration management and input validation in Node.js applications. Latest as of 2024.
