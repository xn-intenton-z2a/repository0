1. Game rules and variations
- Core rule: iterate positive integers. For each integer n:
  - If n divisible by 3 and not 5 -> output "Fizz".
  - If n divisible by 5 and not 3 -> output "Buzz".
  - If n divisible by both 3 and 5 -> output "FizzBuzz".
  - Otherwise output the number n.
- Variations:
  - Additional divisibility rules (e.g., 7) with own tokens.
  - Digit-containing rule: numbers containing digit 3 trigger Fizz behavior even if not divisible by 3; similar for other digits.

2. Minimal implementation pattern for programs
- Loop from 1 to N.
- Check combined divisibility first (n % 15 === 0) to handle 3 and 5 simultaneously; then check 3 and 5 separately.
- Implementation order that works:
  - if (n % 15 === 0) -> "FizzBuzz"
  - else if (n % 3 === 0) -> "Fizz"
  - else if (n % 5 === 0) -> "Buzz"
  - else -> n
- Alternative: build string by concatenation: s = ''; if (n%3===0) s+='Fizz'; if (n%5===0) s+='Buzz'; return s||n
- Performance: trivial O(N) algorithm with constant-time checks per element; memory O(1) for streaming output, O(N) if collecting results.
