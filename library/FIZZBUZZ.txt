1. Definition
FizzBuzz is a deterministic sequence generator: for each integer n in a range, produce either the integer or a substitution token when divisibility conditions match.

2. Rules (precise conditions)
- If n divisible by 3 and 5 (i.e., n % 3 == 0 AND n % 5 == 0) the output is the token "FizzBuzz".
- Else if n divisible by 3 (n % 3 == 0) the output is the token "Fizz".
- Else if n divisible by 5 (n % 5 == 0) the output is the token "Buzz".
- Otherwise output the integer n unchanged.
- Standard domain: positive integers starting at 1. Behavior for 0: 0 % k == 0 so it matches FizzBuzz unless domain excludes 0.

3. Core algorithm (step-by-step)
- Input: N (non-negative integer) and optional start S (default 1).
- For i from S to N inclusive:
  - If i % 15 == 0 then emit "FizzBuzz" (preferred micro-optimization: check combined divisor 15 first).
  - Else if i % 3 == 0 then emit "Fizz".
  - Else if i % 5 == 0 then emit "Buzz".
  - Else emit the integer i.
- Complexity: Time O(N). Memory O(1) if streaming outputs; O(N) if collecting into an array.

4. Parameterized variant (rules map)
- Accept rules: ordered list of (divisor:int, token:string).
- For each i: initialize outToken = empty string; for each rule in rules: if i % rule.divisor == 0 then concatenate rule.token to outToken; after evaluating rules: if outToken non-empty emit outToken else emit i.
- This generalization supports arbitrary divisors and token concatenation (e.g., (3,"Fizz"),(5,"Buzz")) and supports extensible compositions.

5. Edge cases, test vectors
- Input validation: N must be an integer; reject or coerce non-integers explicitly.
- Negative ranges: define semantics (commonly treat as empty sequence or iterate descending).
- Zero: if included, zero is divisible by any non-zero divisor and will produce concatenated tokens; explicitly define whether sequence starts at 1 to avoid unexpected zero output.

6. Supplementary details
- Use combined divisor (LCM) check (e.g., 15 for 3 and 5) to avoid checking two conditions twice; for parameterized rules compute product or LCM as needed when optimization is required.
- For very large N, stream outputs to avoid O(N) memory.
- For parallel or partitioned generation, ensure each partition knows offset S and applies identical rule set deterministically.
