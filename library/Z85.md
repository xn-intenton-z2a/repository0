1. Overview
Z85 is a base-85 encoding defined in ZeroMQ RFC 32 ("32/Z85"). It maps 4 bytes of binary (32 bits) to 5 printable ASCII characters (base-85 digits). The conversion treats each 4-byte block as an unsigned 32-bit integer in network byte order (big-endian) and converts that integer into five base-85 digits; those digits are rendered using the Z85 85-character alphabet.

2. Alphabet (value -> character)
This is the canonical Z85 mapping for values 0..84 (from the spec):

50 - 59: O P Q R S T U V W X

80 - 84: } @ % $ #

3. Encoding algorithm (precise, implementable)
- Input: binary frame whose length is divisible by 4 (application must pad otherwise). Output: printable string whose length is divisible by 5.
- For each 4-octet block (b0,b1,b2,b3) (b0 = first/most-significant byte):
  - n = (b0 << 24) + (b1 << 16) + (b2 << 8) + b3  (unsigned 32-bit big-endian integer)
  - digits = empty array of length 5
  - for i from 4 down to 0:
      digit = n % 85
      digits[i] = digit
      n = floor(n / 85)
    // digits now holds base-85 digits from most-significant (digits[0]) to least-significant (digits[4])
  - for i from 0 to 4: append ALPHABET[digits[i]] to output string
- Repeat for all blocks. The output order is most-significant-digit first.

4. Decoding algorithm (precise)
- Input: string length divisible by 5.
- For each 5-character chunk c0..c4:
  - for i from 0 to 4: value_i = indexOf(ALPHABET, c_i)  (0..84)
  - n = value_0
  - for i from 1 to 4: n = n * 85 + value_i
    // n is now unsigned 32-bit integer
  - output bytes: (n >> 24) & 0xFF, (n >> 16) & 0xFF, (n >> 8) & 0xFF, n & 0xFF
- Concatenate blocks to produce binary frame.

5. Framing and padding rules
- Z85 SPEC: binary frames SHALL have length divisible by 4; string frames SHALL have length divisible by 5. Padding is the responsibility of the application. Do NOT expect built-in padding or special 'z' zero shorthand (Z85 intentionally omits Ascii85's 'z').
- Encoded length calculation: encodedLength = (input.length / 4) * 5
- Decoded length calculation: decodedLength = (encoded.length / 5) * 4

6. Bits per character and UUID sizing
- bits/char = log2(85) = 6.409391 (approx)
- 128-bit (UUID) encoded length = ceil(128 / bits_per_char) = 20 characters (precisely the Z85 block rule gives 16 bytes -> (16/4)*5 = 20 chars)

SUPPLEMENTARY DETAILS

- Interoperability: Z85 uses a fixed 85-character alphabet and strict 4-byte framing to avoid ambiguity between implementations.
- Implementation tips: use unsigned 32-bit arithmetic, ensure input is in network (big-endian) byte order when assembling the 32-bit value (byte[0] is MSB). Validate all input characters are in the Z85 alphabet when decoding.
- Memory/perf: encoding/decoding is linear in the number of 4-byte blocks. Avoid BigInt libraries; all arithmetic fits in 32-bit unsigned range.
