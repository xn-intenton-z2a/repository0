Normalised extract (implementation facts)
- Purpose: Z85 is a deterministic, interoperable base-85 encoding designed for 4-byte chunk safety and source-code friendliness. It encodes 4 bytes -> 5 ASCII characters and disallows ambiguous shorthands.
- Alphabet (index 0..84) exact ordered string (85 characters):
  0123456789abcdefghijklmnopqrstuvwxyzABCDEFGHIJKLMNOPQRSTUVWXYZ.-:+=^!/*?&<>()[]{}@%$#
  (index 0 => '0', index 1 => '1', ... index 10 => 'a', ... index 36 => 'A', ... index 60 => 'Y', index 61 => 'Z', index 62 => '.', ... index 84 => '#')
- Frame length constraints:
  - Binary input frames MUST have length divisible by 4. String output MUST have length divisible by 5. The specification mandates applications to apply their own padding or framing if required; Z85 itself does not define padding.

Encoding algorithm (precise):
- For every 4-octet chunk (bytes b0,b1,b2,b3) assembled as a 32-bit unsigned integer V = (b0<<24) | (b1<<16) | (b2<<8) | b3 (network order):
  - For i from 4 down to 0:
      digit[i] = V % 85
      V = floor(V / 85)
  - Output characters: map digit[4]..digit[0] in most-significant-first order using the alphabet string (i.e., first output char corresponds to the highest-order digit).

Decoding algorithm (precise):
- For every 5 characters, map each char -> value in 0..84 using reverse lookup table. Accumulate V = 0; for each value v: V = V * 85 + v. After 5 chars, emit four bytes: (V >> 24) & 0xff, (V >> 16) & 0xff, (V >> 8) & 0xff, V & 0xff.

Test vector (from spec)
- Binary 8-byte frame:
  86 4F D2 6F B5 59 F7 5B
  Encoded => the ASCII string "HelloWorld" (10 characters). Use this test to validate implementation.

Reference details and API
- encodeZ85(input: Uint8Array) -> string  // requires input length % 4 == 0
- decodeZ85(input: string) -> Uint8Array   // requires input length % 5 == 0
- Implementations MUST validate frame sizes and throw/return errors on mismatched lengths.
- Use a static lookup table of 256 entries for decoding (map unknown chars -> error).

Supplementary details
- Z85 intentionally maps zero bytes into visible characters (leading '0') to keep zero-strings visible in source.
- No padding is defined; for non-4-multiple binary data, callers must pad the binary or use a container format (e.g., length prefix) before encoding.
