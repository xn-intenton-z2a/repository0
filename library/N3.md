Overview and implementation-ready details

DataFactory and term constructors
- DataFactory provides factory constructors according to RDF/JS data model: namedNode(iri), literal(value, languageOrDatatype?), blankNode([id?]), defaultGraph(), quad(subject, predicate, object, graph?)
- Quad structure: {subject: Term, predicate: Term, object: Term, graph: Term}

Parser behavior and options
- N3.Parser parses Turtle/TriG/N-Triples/N-Quads and Notation3 when requested via format option
- Constructor options include:
  - format: 'N-Triples' | 'Turtle' | 'TriG' | 'N3' | MIME type
  - baseIRI: string — base IRI for resolving relative IRIs
  - blankNodePrefix: string — prefix used to avoid blank node collisions
  - isImpliedBy: boolean — controls representation of implication rules
- parse(input, callback) contract:
  - callback(error, quad, prefixes) invoked for each quad; final invocation uses null quad and returns prefixes
  - Alternative object listener form: parse(input, {onQuad, onPrefix, onComment})

StreamParser and streaming
- N3.StreamParser implements Node.js stream and RDF/JS Sink interface; parse streams as they grow and emit quads
- StreamParser supports comments event and prefix event

Writer behavior
- N3.Writer serializes quads into Turtle, TriG, N-Triples, or N-Quads
- Constructor accepts {prefixes: {...}, format: 'N-Triples' | 'application/trig' etc}
- addQuad(quad) / addQuad(subject, predicate, object, graph) and end((error,result) => {...})
- For streaming writers, write to a provided output stream and control end behavior with end(false)

Store and Dataset interface
- N3.Store implements in-memory storage with Dataset interface: attributes and methods:
  - size: integer number of quads
  - add / addQuad / addQuads
  - remove / removeQuad / removeQuads / removeMatches
  - match(subject?, predicate?, object?, graph?) => returns stream or generator/array of matching quads
  - getQuads(subject?, predicate?, object?, graph?) => array of quads
  - countQuads(...)
  - createBlankNode() => BlankNode
- EntityIndex option allows multiple stores to share indexes to reduce memory usage

Supplementary details

Parser strict mode and format option
- Default parser is permissive; pass format option for strict parsing with exact syntax semantics

Writer flags and abbreviation controls
- Flags string can control abbreviations and behavior; combine flags like 'o k' to change abbreviation rules
- Use format option to force specific output syntax (e.g., 'N-Triples' or 'application/trig')

Streaming and memory
- For large files, use StreamParser and StreamWriter to avoid holding whole dataset in memory
- Use N3.Store only for datasets that fit in memory or when a persistent backing store is used
