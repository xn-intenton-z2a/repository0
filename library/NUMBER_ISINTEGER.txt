1. Method signature
- Number.isInteger(value: any) -> boolean

2. Exact behaviour and algorithm
- Returns true only if:
  - typeof value === "number"
  - value is finite (not NaN, +Infinity, -Infinity)
  - Math.floor(value) === value (value has no fractional component)
- Equivalent conditional: return typeof value === 'number' && isFinite(value) && Math.floor(value) === value;

3. Edge cases and examples
- Number.isInteger(1) -> true
- Number.isInteger(1.0) -> true
- Number.isInteger(1.5) -> false
- Number.isInteger(NaN) -> false
- Number.isInteger(Infinity) -> false
- Number.isInteger('1') -> false (no coercion)
- Number.isInteger(0) and Number.isInteger(-0) -> true
- BigInt values: Number.isInteger(1n) -> false because typeof is 'bigint'

4. Polyfill specification
- Polyfill (ECMAScript-consistent): if Number.isInteger is undefined, define as function isInteger(value) { return typeof value === 'number' && isFinite(value) && Math.floor(value) === value; }
- Ensure use of global isFinite with Number.isFinite semantics for numeric type checking.

5. Supplementary details (type/coercion differences)
- Unlike global isFinite or parseInt, Number.isInteger does not coerce its argument; it performs a strict type check first.
- Use when exact numeric-integer tests are required without string/other-type coercion.

6. Reference details (precise conditional expression)
- Implementation-equivalent expression: typeof value === 'number' && isFinite(value) && Math.floor(value) === value
- Signature: Number.isInteger(value) -> boolean
