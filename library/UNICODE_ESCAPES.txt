1. Syntax and flags
- Escape forms: \p{loneProperty}, \P{loneProperty}, \p{property=value}, \P{property=value}
- Requires Unicode-aware RegExp mode: use the u or v flags. v flag extends capabilities (string properties and set notation).
- Flags: g (global), i (ignoreCase), m (multiline), u (unicode), v (regexp v-flag), s (dotAll), y (sticky).

2. Allowed property names and values
- Property names must be ASCII letters and underscores; values must be ASCII letters, underscores, and digits.
- General_Category can be used via short or long forms: Letter, L, Lu, Ll etc.
- Script and Script_Extensions available: use Script=Latin or sc=Latn; Script_Extensions supports scx alias.
- Binary properties can be used as loneProperty; examples: Emoji_Presentation, White_Space.

3. Behavior differences between u and v flags
- u mode: \p and \P match individual code points based on properties; \P is a complement class applied after case-folding when i flag is set.
- v mode: \p may match sequences of code points (properties of strings) such as Emoji sequences (RGI_Emoji_Flag_Sequence). \P complements character properties only and cannot be used for string properties.
- Case-folding differences for complement classes: in u mode, case-folding happens after subtraction; in v mode, case-folding happens before subtraction. This affects patterns like /\P{Lowercase_Letter}/iu vs /\P{Lowercase_Letter}/iv.

4. Examples of common property uses (direct actionable patterns)
- Match all letters from any script: /\p{Letter}/gu or /\p{L}/gu
- Match currency symbols: /\p{Sc}/gu
- Match emoji presentation code points: /\p{Emoji_Presentation}/gu
- Match non-Latin text: /\P{Script_Extensions=Latin}+/gu
- Match emoji flag sequences with v flag: /\p{RGI_Emoji_Flag_Sequence}/v

5. Implementation notes and pitfalls
- Always use u or v flag when using Unicode property escapes; without flags \p is treated as literal 'p'.
- Property names and values must match the ECMAScript Unicode property value aliases (PropertyValueAliases.txt from unicode.org).
- For complex compositions, use v flag set notation intersections and pattern subtraction; alternating behavior across engines requires testing.
- Browser and engine compatibility varies; test critical patterns in target environments (Node versions, V8, SpiderMonkey, JavaScriptCore) and fallback to explicit code point ranges when necessary.

SUPPLEMENTARY DETAILS

- Use Unicode property escapes to avoid brittle ranges; they are broader and handle non-Latin scripts correctly.
- For matching punctuation or digits across scripts, use General_Category values: P (punctuation), N (number), Sc (currency symbol).

REFERENCE DETAILS (specifications and exact patterns)

- Syntax: \p{property=value} or \p{loneProperty}
- Examples: \p{Script=Grek}, \p{sc=Cyrl}, \p{General_Category=Number} (or \p{N})
- Flags required: u or v. Use 'v' for properties of strings and set notation.
- RegExp constructors: new RegExp('\p{Letter}','gu') is invalid unless using proper escaping in JS string literals; prefer /\p{Letter}/gu in source code.

BEST PRACTICES

- Prefer property escapes over manual ranges when supporting internationalized input.
- For emoji handling, use v flag to match multi-code-point sequences (emoji ZWJ sequences, flag sequences).
- Validate property names against official Unicode aliases to avoid runtime syntax errors.

TROUBLESHOOTING

- SyntaxError: "Invalid property name" indicates unsupported or misspelled property/value; verify against PropertyValueAliases.txt.
- Unexpected mismatches across browsers: check whether the engine supports v flag or string properties; if not, fall back to u mode or explicit sequences.
