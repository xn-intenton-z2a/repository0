NORMALISED EXTRACT
1. Rules and canonical behaviour
- Sequence: integer sequence starting at 1 counting upwards. For each integer i:
  - If i divisible by both 3 and 5 -> string "FizzBuzz".
  - Else if i divisible by 3 -> string "Fizz".
  - Else if i divisible by 5 -> string "Buzz".
  - Else -> decimal representation of i as a string.
- Divisibility implemented via integer modulus: i % 3 === 0, i % 5 === 0; combined check i % 15 === 0 recommended for correctness and minimal branches.

2. Implementation patterns (JS)
- Basic function pattern:
  - Iterate i from 1 to N inclusive with a single for-loop.
  - For each i evaluate i % 15 === 0 first, then i % 3 === 0, then i % 5 === 0, otherwise push String(i).
  - Return an array of strings length N where array[index] corresponds to value for i = index+1.
- Alternative: use generator function to yield values lazily for large ranges: function* fizzBuzzGen(limit) { for (let i=1;i<=limit;i++){ ... yield out; }}
- Avoid repeated string concatenation per iteration; use fixed literals.

3. Input validation and errors
- Use Number.isInteger(value) to determine if input is integer. Number.isInteger(x) returns true for numeric integer values and false for NaN, Infinity, non-number types, and non-integer floats.
- Validate range: require positive integers (n >= 1). If invalid, throw RangeError with clear message, e.g., "n must be an integer >= 1". Follow MDN RangeError constructor semantics: throw new RangeError(message).
- Protect against excessive memory by capping N or use streaming/generator when N may be large.

4. Module/export patterns for distribution
- Use ES module exports: export function fizzBuzz(n) { ... } and import via import { fizzBuzz } from './fizz.js'. Ensure environment supports modules via one of:
  - package.json contains "type": "module" to allow .js as ESM.
  - Use .mjs extension and serve with correct MIME type text/javascript.
  - Use <script type="module"> in browser contexts.
- For browser bare imports use an import map mapping bare specifiers to URLs.

5. Testing with Vitest
- Install: npm install -D vitest. Vitest requires Node >=20 and Vite >=6.
- Test files must include .test. or .spec. in filename. Example test flow:
  - Import fizzBuzz function into test file.
  - Write assertions for small N: fizzBuzz(1) -> ["1"], fizzBuzz(3) -> ["1","2","Fizz"], fizzBuzz(5) includes "Buzz" and at 15 includes "FizzBuzz".
- Run: npm run test or npx vitest --run. For CI use vitest run --coverage.
- Vitest config: use vitest.config.(js|ts) or define test property in vite.config; tests run in Node by default unless environment changed.

6. Performance and streaming
- For N up to a few million prefer generator or streaming output rather than building huge arrays to reduce peak memory.
- Time complexity O(N), space O(1) for generator output, O(N) for array output.
- Avoid repeated expensive operations inside loop; precompute divisibility check order and use integer arithmetic only.
