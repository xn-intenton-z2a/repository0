1. Module file types and server MIME requirements
- Module files may use .js or .mjs extensions. Node and many tools accept .mjs to indicate ESM explicitly; browsers require correct Content-Type header when serving modules (.js or .mjs).
- Servers must serve module files with a JavaScript MIME type such as text/javascript. Failure to serve correct MIME results in a strict MIME type checking error and the browser will not run the script.
- For portability: prefer .js with either <script type="module"> in browser or package.json "type":"module" in Node projects to avoid .mjs server MIME issues.

2. Export syntax and export styles
- Named export inline: export const name = "square";
- Named export function: export function draw(ctx, length, x, y, color) { ... }
- Aggregate export at file end: export { name, draw, reportArea, reportPerimeter } to control public API.
- Default export: export default function main() { ... } (imported without braces).
- Exports must be top-level; cannot be used inside function scopes.

3. Import syntax and module specifiers
- Named import: import { name, draw } from "./modules/square.js";
- Default import: import whatever from './module.js';
- Module specifiers resolve relative to document base for browsers; use ./ and ../ prefixes for relative imports to ensure portability.
- Imported bindings are live read-only views: you cannot reassign imported binding names but can mutate exported object properties if they are objects.

4. Import maps and remapping rules
- Import map structure (script type="importmap") maps bare specifier keys to absolute or relative URLs. Keys without trailing slash are exact matches; keys with trailing slash act as path prefixes.
- When multiple keys match a specifier, the longest matching key is chosen.
- Example imports in importmap replace module specifiers at resolution time allowing bare names in browsers (e.g., import {draw} from "square" when "square" maps to "./shapes/square.js").
- Import maps only apply to the containing document; they are not automatically applied to workers or worklets.

5. Feature detection and environment notes
- Detect import map support via HTMLScriptElement.supports?.("importmap").
- Use <script type="module"> to load modules in browsers which provides module semantics (deferred execution, strict mode, top-level await support in modern browsers).
- Node ESM: enable by using package.json "type":"module" or .mjs extensions; Node resolution rules differ from browser resolution (package exports, package.json "exports" field).
