1. Project overview and purpose
- node-canvas is a Cairo-backed Canvas implementation for Node.js implementing the Web Canvas API as closely as possible, including utilities for PNG/JPEG/PDF/SVG output and font registration.

2. Installation and compiling requirements
- npm install canvas (prebuilt binaries downloaded for common platforms)
- Minimum Node.js: 18.12.0
- To build from source, dependencies include Cairo (>=1.10.0), Pango, libpng, libjpeg, giflib, librsvg; platform-specific instructions provided on the repo wiki.

3. Core factory and utility methods (literal signatures from source)
- createCanvas(width: number, height: number, type?: 'PDF'|'SVG') => Canvas
- createImageData(width: number, height: number) => ImageData
- createImageData(data: Uint8ClampedArray|Uint16Array, width: number, height?: number) => ImageData
- loadImage(src) => Promise<Image>
- registerFont(path: string, { family: string, weight?: string, style?: string }) => void
- deregisterAllFonts() => void

4. Canvas encoding and stream methods (from source)
- canvas.toBuffer([callback], [mimeType], [config]) => Buffer | void (callback) with mime types: 'image/png','image/jpeg','raw','application/pdf','image/svg+xml'
- canvas.createPNGStream(config?: any) => ReadableStream
- canvas.createJPEGStream(config?: any) => ReadableStream
- canvas.createPDFStream(config?: any) => ReadableStream
- canvas.toDataURL(... variations supported) => data URL or callback

5. Context quality and text rendering options
- context.patternQuality: 'fast'|'good'|'best'|'nearest'|'bilinear' (defaults to 'good')
- context.quality: same options as patternQuality (defaults to 'good')
- context.textDrawingMode: 'path'|'glyph' (defaults to 'path') with rendering implications for image/PDF/SVG canvases

6. Non-standard APIs highlighted
- Image#src accepts Buffer, local path, remote URL, or data: URI; use img.onload and img.onerror for asynchronous handling
- Image#dataMode controls MIME/image tracking for PDF canvases (Image.MODE_MIME, Image.MODE_IMAGE)
- canvas.toBuffer config options for jpeg/png/pdf are described (quality, progressive, chromaSubsampling, compressionLevel, filters, resolution, PDF metadata fields)
