3. Integer to Roman conversion algorithm (toRoman)
- Use ordered parallel arrays:
  values = [1000, 900, 500, 400, 100, 90, 50, 40, 10, 9, 5, 4, 1]
  symbols = ["M", "CM", "D", "CD", "C", "XC", "L", "XL", "X", "IX", "V", "IV", "I"]
- Algorithm (imperative greedy loop):
  - Initialize result = empty string, index i = 0.
  - While n > 0:
    - times = Math.floor(n / values[i])
    - If times > 0, append symbols[i] repeated times to result, and subtract values[i] * times from n.
    - Increment i.
  - Return result.
- Produces standard subtractive Roman numerals (e.g., 4 -> IV, 9 -> IX, 40 -> XL, 90 -> XC, 400 -> CD, 900 -> CM).

4. Roman to integer conversion algorithm (fromRoman)
- Precondition: str is trimmed and uppercased, validated against canonical regex.
- map = { I: 1, V: 5, X: 10, L: 50, C: 100, D: 500, M: 1000 }
- Iterative parse with lookahead:
  - total = 0
  - Iterate i from 0 to str.length - 1:
    - value = map[str[i]]
    - next = map[str[i+1]] if exists else 0
    - If next > value:
      - total += next - value
      - i += 1 (skip the next char)
    - Else:
      - total += value
  - Return total
- This algorithm relies on prior strict validation to guarantee canonical subtractive pairs and prevent ambiguous/nonstandard inputs.

5. Validation regex and canonical form rules
- Canonical Roman numeral regex used to validate 1..3999 strictly:
  ^M{0,3}(CM|CD|D?C{0,3})(XC|XL|L?X{0,3})(IX|IV|V?I{0,3})$
- Rules enforced by regex:
  - Thousands: 0..3 M characters only (0..3000)
  - Hundreds: allow 900 (CM), 400 (CD), or D?C{0,3} (0..800)
  - Tens: allow 90 (XC), 40 (XL), or L?X{0,3}
  - Units: allow 9 (IX), 4 (IV), or V?I{0,3}
- This enforces canonical subtractive notation and prevents invalid forms like IIV, VX, or nonstandard repetitions.

7. Edge cases and allowed range
- Allowed numeric input: integer in range 1..3999 inclusive.
- toRoman(0) or toRoman(4000) should raise RangeError.
- fromRoman must accept mixed-case and whitespaced input by trimming and uppercasing, but still rejects non-canonical forms.
- Empty or non-string inputs to fromRoman throw an error.
- The implementation assumes standard Roman numeral conventions and does not support overline (multiples of 1000 beyond 3999).

8. Complexity and performance
- toRoman: O(k) where k is number of symbol groups appended; in worst case k is proportional to number (bounded constant because max 3999), effectively O(1) for this domain.
- fromRoman: O(L) where L is length of the input string (<= 15 for canonical 3999 as 'MMMCMXCIX'), so effectively O(1) and very fast.
