1. Installation and environment requirements
- Install with npm: npm install -D vitest (or yarn/pnpm/bun equivalents).
- Requirements: Vite >= 6.0.0 and Node >= 20.0.0.
- Recommended to add vitest to devDependencies in package.json and use npx vitest to run if not installed locally.

2. Writing and naming tests
- Default test file name pattern: must include .test. or .spec. in filename.
- Example test imports: import { expect, test } from 'vitest'; test('description', () => { expect(...).toBe(...) })
- Tests can be authored alongside source or in dedicated test directories; ensure file globs match config.

3. Configuration options and project support
- Vitest reads vite.config.ts; if vitest specific config needed, create vitest.config.ts or pass --config.
- Supported config file extensions: .js, .mjs, .cjs, .ts, .cts, .mts (not .json).
- Projects support: define multiple project configs under test.projects in vitest config to run varied environments (node, happy-dom, etc.).
- Automatic dependency installation can be disabled with VITEST_SKIP_INSTALL_CHECKS=1.

4. CLI usage and common scripts
- Typical package.json scripts: "test": "vitest", "coverage": "vitest run --coverage".
- To run once without watch: vitest run; to run in watch mode: vitest (without run flag).
- Use --config to point to specific config file; use environment variables (VITEST or import.meta.env) to conditionally alter config.

SUPPLEMENTARY DETAILS

Technical specifications and implementation details:
- Vitest integrates with Vite transformation pipeline; when using, keep Vite plugins and resolve.alias consistent between dev and test by sharing config or merging via mergeConfig.
- Projects support allows running different test configs in the same process by providing list of config globs or config objects.

REFERENCE DETAILS

API signatures and configuration options:
- Test API: test(name: string, fn: Function) -> void
- Assertion API: expect(value).toBe(expected) and other matchers (see Vitest API reference).
- Config: defineConfig({ test: { /* options */ } }) from 'vitest/config'; test object accepts projects, environment, setupFiles, name, root, etc.

Troubleshooting and best practices:
- If tests fail due to missing transforms: ensure Vite plugins used in app are available to Vitest via vite.config or vitest.config.
- If dependency prompts are undesired in CI: set VITEST_SKIP_INSTALL_CHECKS=1.
- For TypeScript projects: reference Vitest types in config via triple-slash directive /// <reference types="vitest/config" /> at top of vite.config.ts when using vitest-specific typings.
