1. CLI usage and scripts
- Run tests: vitest --run [pattern]
- Common npm script: "test": "vitest --run tests/unit/*.test.js"
- For coverage: use @vitest/coverage-v8 plugin and run vitest with coverage flags/config.

2. Test API (describe, test/it, hooks)
- describe(name: string, fn: () => void) -> void
- test(name: string, fn: () => Promise|void) -> void
- it is an alias for test
- Hooks: beforeEach(fn), afterEach(fn), beforeAll(fn), afterAll(fn) — each accepts synchronous or async functions.

3. Expect API and common matchers
- expect(received).toBe(expected)  // strict equality
- expect(received).toEqual(expected) // deep equality
- expect(received).toBeTruthy()/toBeFalsy()
- expect(mock).toHaveBeenCalled()/toHaveBeenCalledWith(...args)
- Async assertions: await expect(promise).resolves.toEqual(value); await expect(promise).rejects.toThrow(error)

4. Mocking utilities (vi)
- vi.fn(implementation?) -> mock function
- vi.spyOn(object, 'method') -> spy that can be stubbed
- vi.mock(modulePath, factory?) -> mock module
- vi.restoreAllMocks() -> restore spies and mocks

5. Timer control and async support
- vi.useFakeTimers() to replace real timers
- vi.advanceTimersByTime(ms: number) to fast-forward fake timers
- Supports top-level async tests and returning Promises from test functions

6. Configuration and coverage integration
- Configure vitest via package.json or vitest.config.js
- Use "--run" for non-watch CI execution
- Integrate coverage through @vitest/coverage-v8 and configure reporters/options in vitest config

7. Best practices and troubleshooting
- Ensure project module type aligns with tests (type: "module" in package.json when using ESM).
- For ESM modules, ensure import paths include file extensions or package exports are defined.
- If tests fail due to module resolution, run vitest with increased logging or check node "type" and file extensions.

8. Reference details (function signatures and options)
- describe(name: string, fn: () => void) -> void
- test(name: string, fn: () => Promise|void, timeout?: number) -> void
- expect(value: any) -> Expectation object with matcher methods
- vi.fn(impl?: Function) -> MockFunction
- vi.spyOn<T, K extends keyof T>(obj: T, key: K) -> Spy
