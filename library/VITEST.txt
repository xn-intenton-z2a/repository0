1. Installation and runtime requirements
- Install with npm install -D vitest or yarn add -D vitest.
- Requires Vite >= 6.0.0 and Node >= 20.0.0 when used with Vite transformations.

2. Basic test structure and conventions
- Test files should include .test. or .spec. in their filename by default.
- Basic API: import { test, expect } from 'vitest'
- Example test: test('adds', () => { expect(sum(1,2)).toBe(3) })
- Exported test runner supports both watch and run modes; use vitest run for single-run CI.

3. Configuration and integration with Vite
- Vitest reads root vite.config.* if present; specific vitest.config.* overrides this.
- Config file supports export default defineConfig({ test: { /* options */ } }) and shares Vite settings like resolve.alias.
- To merge configs: use mergeConfig from 'vitest/config' to merge Vite and Vitest configs.

4. CLI and common commands
- npm script examples: "test": "vitest", "coverage": "vitest run --coverage".
- Run in single-shot CI: vitest run --reporter dot or other reporters.
- Disable auto dependency installer with VITEST_SKIP_INSTALL_CHECKS=1.

5. Projects and multi-config setups
- vitest.config test.projects accepts array of project definitions or globs to run multiple environments.
- Use projects to run the same tests under different environments (node, happy-dom, browser-like environments) with separate setupFiles.

6. Recommended practices and troubleshooting
- Keep vitest and vite config consistent; vitest config overrides vite config when both present for keys under test.
- For TypeScript projects add triple-slash directive to config for type references: /// <reference types="vitest" />
- If automatic dependency prompts are undesired in CI, set VITEST_SKIP_INSTALL_CHECKS=1.
- When using Bun, run tests with bun run test instead of bun test to avoid Bun's test runner being invoked.

SUPPLEMENTARY DETAILS

Implementation specifics
- Test projects example structure: provide separate environment names, root, and setupFiles per project in test.projects.
- Vitest relies on Vite's transform pipeline; for non-Vite projects provide vitest.config.ts with test property and necessary plugins.

REFERENCE DETAILS

Configuration options and effects (selected)
- test.environment: 'node' | 'happy-dom' | 'jsdom' etc. Controls global DOM availability and APIs.
- test.setupFiles: string[] — files to run before tests to initialize globals.
- test.include/exclude: globs to include or skip test files.

CLI options (common)
- vitest [options]
  - run: execute tests once (CI)
  - --coverage: collect coverage
  - --config <path>: use alternate config file

BEST PRACTICES
- Install vitest locally as dev dependency to ensure consistent runner version in CI.
- Share Vite config where possible; use mergeConfig if separate files must exist.
- Use projects to run the same code in node and browser-like environments when necessary.

TROUBLESHOOTING

Common issues
- Missing dependencies prompt: prevent in CI by setting VITEST_SKIP_INSTALL_CHECKS=1.
- Conflicting configs: verify which file is being read; vitest.config.* has priority over vite.config for test keys.
