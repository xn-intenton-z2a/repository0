1. Signature and parameters
codePointAt(index)
Parameter: index - zero-based integer index into the string; converted to integer (undefined -> 0).

2. Return value and range
Returns a non-negative integer representing the Unicode code point value of the character at the given index. Valid code points: 0 to 0x10FFFF.
- If index out of range (index < 0 or index >= str.length) the method returns undefined.

3. Behavior with UTF-16 surrogate pairs
- If the element at index is a UTF-16 leading surrogate and it forms a valid surrogate pair with the following code unit, codePointAt returns the combined code point of the surrogate pair (the full Unicode code point > 0xFFFF).
- If the element at index is a UTF-16 trailing surrogate, codePointAt returns the numeric value of that single surrogate code unit (not a combined code point).

4. Looping and iteration guidance
- Avoid iterating strings by numeric indices in presence of surrogate pairs because codePointAt will be invoked twice for a surrogate pair when looping indices; prefer iterating by code points.
- Use for...of or [...str] to obtain code point elements (each element is a string representing one Unicode code point or code point plus modifiers). Then use .codePointAt(0) on each element to get numeric value.
Example pattern:
  for (const cp of str) { const value = cp.codePointAt(0); }

5. Specification links
- ECMAScript® specification: sec-string.prototype.codepointat (text-processing section)

6. Supplementary details
- codePointAt is part of the String.prototype API; it differs from charCodeAt() which returns a UTF-16 code unit (0..65535) and cannot represent code points above 0xFFFF as a single value.
- When working with grapheme clusters (user-perceived characters that may be multiple code points) codePointAt and Symbol.iterator iterate code points, not grapheme clusters; further normalization or grapheme segmentation libraries are required when grapheme-aware iteration is needed.

7. Reference details
- Signature: String.prototype.codePointAt(index) -> number | undefined
- Parameter conversion: index is ToInteger(index) per ECMAScript rules; undefined coerces to 0.
- Behavior: If code unit at index is a high surrogate and next code unit is a low surrogate, return ((high - 0xD800) << 10) + (low - 0xDC00) + 0x10000; else return code unit value.
