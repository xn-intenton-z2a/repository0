1. CSV format core rules (literal rules as defined in RFC 4180 section 2)
- Each record is on a separate line, delimited by CRLF. The last record may or may not have a trailing CRLF.
- Optional header line may appear as the first line; header fields must match the number of fields in records.
- Fields are separated by commas; each line should contain the same number of fields throughout the file.
- Spaces are part of a field and must not be ignored; the last field must not be followed by a comma.
- Fields may be enclosed in double quotes. If not enclosed, double quotes cannot appear inside fields.
- Fields containing line breaks (CRLF), double quotes, or commas must be enclosed in double quotes.
- Double quotes inside an enclosed field must be escaped by preceding them with another double quote (""").

2. ABNF grammar (literal extract)
file = [header CRLF] record *(CRLF record) [CRLF]
header = name *(COMMA name)
record = field *(COMMA field)
name = field
field = (escaped / non-escaped)
escaped = DQUOTE *(TEXTDATA / COMMA / CR / LF / 2DQUOTE) DQUOTE
non-escaped = *TEXTDATA
COMMA = %x2C
CR = %x0D
LF = %x0A
CRLF = CR LF
DQUOTE = %x22
TEXTDATA = %x20-21 / %x23-2B / %x2D-7E

3. MIME registration details
- Registered MIME type: text/csv
- Optional parameters: charset and header (header valid values: "present" or "absent").
- Encoding considerations: CRLF used for line breaks but implementations may vary.

4. Optional parameters and encoding considerations
- "charset" parameter can specify character set (commonly US-ASCII but others acceptable).
- "header" parameter indicates the presence or absence of a header line; implementors can choose defaults if not present.

5. Security and interoperability notes
- CSV files are passive text but may include malicious binary data to exploit buffer overruns; validate inputs.
- Interoperability: wide divergence exists; implementors should be liberal in parsing and conservative in output.

6. Reference details (exact config values and grammar available above)
- Use the ABNF grammar as the authoritative parsing specification.
- Treat fields with embedded CRLF, COMMA, or DQUOTE as escaped with DQUOTE and internal DQUOTE doubled.
