1. Definition
Hamming weight (population count, popcount) of a bitstring or integer is the number of bits set to 1. For an integer x, weight(x) = number of 1 bits in binary representation.

2. Relationship to Hamming distance
For binary strings a and b, HammingDistance(a,b) = popcount(a XOR b). Thus efficient popcount implementations enable fast Hamming distance calculations on integer representations.

3. Efficient algorithms and constants (explicit details)
- Wegner / Kernighan method: loop while x != 0: x &= x - 1; count++ ; complexity proportional to number of 1 bits.
- Parallel-add / tree methods (word-level): combine bit sums using masks and shifts. Constants for 64-bit implementations:
  m1  = 0x5555555555555555  // 0101...
  m2  = 0x3333333333333333  // 00110011...
  m4  = 0x0f0f0f0f0f0f0f0f  // 4-bit groups
  m8  = 0x00ff00ff00ff00ff  // 8-bit groups
  m16 = 0x0000ffff0000ffff  // 16-bit groups
  m32 = 0x00000000ffffffff  // 32-bit groups
  h01 = 0x0101010101010101  // multiplier for horizontal sum

Example algorithm forms (pseudocode):
- popcount64a (tree-addition):
  x = (x & m1) + ((x >> 1) & m1)
  x = (x & m2) + ((x >> 2) & m2)
  x = (x & m4) + ((x >> 4) & m4)
  x = (x & m8) + ((x >> 8) & m8)
  x = (x & m16) + ((x >> 16) & m16)
  x = (x & m32) + ((x >> 32) & m32)
  return x

- popcount64b (fewer ops):
  x -= (x >> 1) & m1
  x = (x & m2) + ((x >> 2) & m2)
  x = (x + (x >> 4)) & m4
  x += x >> 8
  x += x >> 16
  x += x >> 32
  return x & 0x7f

- popcount64c (multiply-based):
  x -= (x >> 1) & m1
  x = (x & m2) + ((x >> 2) & m2)
  x = (x + (x >> 4)) & m4
  return (x * h01) >> 56

- popcount64d (Wegner):
  count = 0
  while x != 0:
    x &= x - 1
    count += 1
  return count

4. Implementation signatures
- C-style: int popcount64a(uint64_t x) -> returns number of 1 bits (0..64)
- Alternative: __builtin_popcount(unsigned) and __builtin_popcountll(unsigned long long) available on GCC/Clang; return type int.

5. Memory-time tradeoffs and processor support
- Lookup table approach: store popcount for 16-bit words in table[65536], compute 32-bit popcount by two table lookups and addition.
- Hardware support: many CPU ISAs expose POPCNT/POPC/VCNT/CPOP instructions; use compiler intrinsics when available for best performance.
- Vectorized and Harley–Seal algorithms can outperform scalar implementations on platforms with SIMD.

SUPPLEMENTARY DETAILS
- Use Wegner/Kernighan when sparse bits expected; use tree/multiply methods or intrinsics when dense or when constant-time behavior desired.
- For large bit vectors, process in machine-word chunks and sum per-word popcounts.
