3. Agentic pipeline: flow and responsibilities (concrete)
- Input: MISSION.md (plain-English project goals).
- Supervisor agent reads repository context (issues, PRs, workflow runs, features).

- Workflow cycle: MISSION.md → supervisor → dispatch workflows → create Issue → code changes → run Tests → create PR → merge (on success).
- init.yml runs daily to keep agentic infrastructure updated; workflows run according to schedule configured in agentic-lib.toml.
