1. Purpose and scope
- Vega-Lite is a high-level declarative grammar for interactive graphics.
- It compiles concise JSON specifications to Vega, which handles rendering to SVG/PNG.

2. Specification model (declarative JSON grammar)
- Specifications describe mappings from data to properties of graphical marks (e.g., point, bar).
- The compiler auto-produces axes, legends, and scales; sensible defaults are applied but can be overridden.

3. Visualization primitives and encodings
- Mark types: point, bar, line, area, etc. (Vega-Lite standard marks; exact list available in docs).
- Encodings map data fields to visual channels: x, y, color, size, shape, opacity, etc.

4. Data transformations supported
- Aggregation, binning, filtering, sorting.
- Visual transformations: stacking, faceting.

5. Composition and interactivity
- Compositions: layering, concatenation, small multiples (facet), and repeated views.
- Interactivity via selections (point/interval selections) to link views and enable brushing/zooming/filtering.

6. Supplementary implementation details
- Vega-Lite compiles to Vega JSON for low-level rendering control.
- JSON schema is provided and versioned; latest schema link referenced: https://vega.github.io/schema/vega-lite/v6.json

7. Reference details (direct actionable specs and patterns)
- Author a JSON object with top-level keys: $schema (schema url), data (values|url|name), mark, encoding, transform, selection, layer/concat/facet.
- Example minimal spec pattern: { "$schema": ".../vega-lite/v6.json", "data": {"values": [...]}, "mark": "bar", "encoding": {"x": {"field":"...","type":"quantitative"}, "y": {...}} }
- Use the compiler to convert Vega-Lite to Vega if needing lower-level customization.
