1. Named export syntax
- Exact form: export { name1, name2 as alias } ;
- Named export binds the referenced local binding to the module's exported namespace. The exported binding is a live read-only view: importing modules observe updates to the original binding's value.

2. Default export syntax
- Exact form: export default <expression> ; or export default function f() { } or export default class C { } ;
- Default export provides a single unnamed/default binding per module; importers can import it with any local name.

3. Re-exporting and export-from
- Re-export select bindings from another module: export { name1, name2 as alias } from 'module';
- Re-export all public bindings: export * from 'module';
- Re-export default explicitly: export { default as alias } from 'module';
- Re-export does not create a local binding in the current module by default; it forwards bindings from the source module.

4. Live bindings and evaluation semantics
- Exports are live bindings: if exported local binding changes after module initialization, importers reading the binding will observe the new value.
- Module evaluation order: imports are resolved and modules are linked before executing module bodies; circular dependencies expose partially-initialized bindings but live bindings allow updates to propagate.

5. Export grammar forms and effects
- Named list: export {name1, name2 as alias}; effect: named exports added to module namespace.
- Declaration export: export function f() {} or export const x = 1; effect: declares local binding and exports it.
- Default: export default expr; effect: sets module's default export slot.
- Re-export: export {x} from 'mod'; effect: re-exports binding from mod without creating a local binding unless explicitly imported locally.

6. Supplementary details (best practices)
- Prefer named exports for libraries (clear ABI, tree-shaking friendly).
- Use default export for single-primary-value modules (but be aware of refactoring friction).
- Avoid circular re-export chains that complicate initialization order.
