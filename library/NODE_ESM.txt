1. Enabling ESM in Node.js
- Mark JS files as ESM by any of:
  - Use .mjs file extension.
  - Set package.json field "type": "module" for .js files to be treated as ESM.
  - Use CLI flag --input-type=module for inline evaluation.
- Conversely, .cjs or "type":"commonjs" marks CommonJS.
- When undecorated, Node will inspect source for ESM syntax to choose system.

2. Import specifiers and resolution rules
- Specifier types:
  - Relative: './file.js' or '../x.mjs' — file extension mandatory for import keyword; directory indexes must be fully specified.
  - Bare specifiers: 'package' or 'package/path' — resolved via Node package resolution; file extension may be required when package lacks "exports" field.
  - Absolute: file:///path/to/file.js
- Mandatory file extensions: import statements require explicit file extensions for relative/absolute specifiers (to match browser behavior).
- Bare specifier resolution follows Node's package resolution and respects package.json "exports" mappings.

3. import.meta properties and APIs
- import.meta.url: string file: URL of the current module.
- import.meta.filename: absolute file path (symlinks resolved); only for file: modules.
- import.meta.dirname: directory name for file modules.
- import.meta.main: boolean indicating module is entry point (stability early development).
- import.meta.resolve(specifier[, parentURL]) -> string absolute URL that specifier would resolve to; synchronous and may perform filesystem ops; parentURL optional when experimental.

4. Import attributes and URL schemes
- Node supports import attributes with only the type attribute; type:'json' mandatory for JSON modules.
- Supported URL schemes: file:, node:, data: (with MIME constraints for data: imports), and node: for builtin modules.
- data: imports work with text/javascript for JS, application/json for JSON, application/wasm for Wasm.

SUPPLEMENTARY DETAILS

Implementation and interoperability guidance:
- Use url.pathToFileURL when converting filesystem paths to import specifiers to avoid percent-encoding errors.
- For dynamic import(), use await import(specifier) which returns a Promise resolving to module namespace.
- Built-in modules can be imported with node: prefix to avoid name collisions.

REFERENCE DETAILS

Exact behaviors and constraints:
- import resolution: Relative/absolute require explicit extensions; import() supports both static and dynamic use and interoperates with CJS.
- import.meta.resolve(specifier) signature: returns string absolute URL; may accept parentURL (string|URL) when experimental; synchronous behavior.
- JSON import rule: import foo from './file.json' with { type: 'json' } or via dynamic import with with option.
