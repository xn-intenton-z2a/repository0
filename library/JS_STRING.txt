1. String iteration and code points
JavaScript strings are sequences of UTF-16 code units. Characters outside the BMP (code points > 0xFFFF) are represented as surrogate pairs (two 16-bit code units). To iterate logical Unicode code points, use the string iterator (for...of) or spread syntax which yield full code point strings (surrogate pairs preserved as single iteration values). Index-based iteration (for i = 0; i < str.length; i++) walks code units and may split surrogate pairs.

2. String.prototype.codePointAt(index)
Signature: codePointAt(index) -> Number | undefined
Parameters: index (converted to integer); undefined converts to 0.
Return value: a non-negative integer in range 0..1114111 representing the Unicode code point at the given index. If index out of range, returns undefined. If the element at index is a UTF-16 leading surrogate, returns the code point formed by the surrogate pair; if it is a trailing surrogate, returns the trailing surrogate code unit (not the combined code point).
Notes: When iterating by indices over a string containing surrogate pairs, codePointAt will return the paired code point only on the leading surrogate index; on the trailing surrogate index it returns the trailing surrogate code unit.

3. String[Symbol.iterator]()
Signature: string[Symbol.iterator]() -> iterable iterator
Return value: an iterable iterator that yields Unicode code points of the string as individual strings, preserving surrogate pairs and yielding each code point once. The iterator yields string elements each corresponding to one code point (may be length 1 or 2 in underlying UTF-16 units).

4. Practical patterns for Unicode-aware Hamming computations
- To compute Hamming distance between two user-visible strings (by code point), convert both to arrays of code point strings via [...str] or Array.from(str) (both use the string iterator), then compare element-by-element, ensuring equal length after conversion.
- Pattern (conceptual):
  a_cp = [...a]; b_cp = [...b];
  if a_cp.length != b_cp.length: error
  for i in range(a_cp.length): compare a_cp[i] !== b_cp[i]
- When you need numeric code point values for operations, for-of or spread yields codePointStrings; call codePointAt(0) on each yielded string to get the numeric code point.

SUPPLEMENTARY DETAILS
- MDN notes: prefer for...of or spread when working with Unicode code points to avoid surrogate-splitting bugs. codePointAt exists on ES2015+ and is in ECMAScript spec.
