1. Function signatures
- _.camelCase([string='']): string
- _.kebabCase([string='']): string

2. Behavior rules and input handling
- Both functions coerce input to string. Nullish or non-string values are converted via String(value).
- They normalize separators (spaces, underscores, dashes, periods, and various punctuation) into word boundaries.
- They remove non-word characters and collapse multiple separators.
- camelCase: transforms word sequence into lowercased-first-word followed by capitalized subsequent words, removes separators. Example behaviour: "Foo Bar" -> "fooBar".
- kebabCase: transforms into lowercased words joined by hyphens, e.g., "Foo Bar" -> "foo-bar".

3. Tokenization and separator rules
- Word tokens are detected by: sequences of letters/digits, transitions from lower to upper case within words (e.g., "fooBar" -> ["foo","Bar"]), numeric runs, and Unicode letter categories where supported.
- Adjacent punctuation and whitespace are treated as separators; multiple separators collapse into one boundary.
- Leading and trailing separators are ignored.

4. Edge cases and examples
- Strings with multiple uppercase letters: "XMLHttpRequest" is typically tokenized to ["XML","Http","Request"] resulting in camelCase -> "xmlHttpRequest".
- Empty input yields empty string output.
- Non-Latin scripts: behavior depends on underlying Unicode support of the tokenization; Lodash uses regex heuristics that may vary in cross-language ports.

5. Implementation notes for porting
- To port behavior, implement Unicode-aware tokenization: use Unicode property escapes (\p{Letter}, \p{Number}) where available; fallback to manual ranges otherwise.
- For camelCase implement: split into tokens, lower-case first token, capitalize first code point of subsequent tokens, then join without separators. For capitalization, use locale-insensitive toLowerCase/toUpperCase when locale not specified; for locale-aware casing, accept optional locale parameter.
- For kebabCase implement: split tokens, lower-case all tokens, join with '-'.

SUPPLEMENTARY DETAILS

- Performance: tokenization uses regex scanning; ensure compiled patterns reused for repeated calls.
- Normalisation: trim and replace diacritics only if desired; Lodash does not decompose diacritics by default.

REFERENCE DETAILS (API signatures and implementation patterns)

- camelCase input -> String(value) -> tokens[] -> first = tokens[0].toLowerCase() -> for i>0: tokens[i] = capitalize(tokens[i]) -> output = first + tokens[1..].join('')
- kebabCase input -> String(value) -> tokens[] -> tokens = tokens.map(t => t.toLowerCase()) -> output = tokens.join('-')

BEST PRACTICES

- Reuse compiled regexes for tokenization to avoid allocation overhead.
- If supporting internationalized casing, consider optional locale parameter and use Intl APIs for correct results.

TROUBLESHOOTING

- Unexpected capitalisation for acronyms: ensure split rule treats sequences of uppercase followed by lowercase as separate token boundary.
- Mixed-script inputs: verify tokenization handles script transitions; if incorrect, add explicit script-based rules or use Unicode property escapes.
