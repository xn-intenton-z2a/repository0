1. Signature and semantics
- Function: Number.isInteger(value)
- Parameter: value — any JavaScript value to be tested for being an integer.
- Return: boolean — true if value is a number value that is an integer, otherwise false.

2. Return conditions and edge cases
- Returns false for NaN, Infinity, -Infinity.
- Returns true for numeric values that are mathematically integers and representable in IEEE-754; due to floating-point precision, some values like 5.0000000000000001 may be treated as 5 and return true.
- For values outside Number range or non-number types (strings, booleans, objects), returns false.
- Loss of precision near Number.MAX_SAFE_INTEGER may cause non-integers to appear integer; validate where exact integer semantics required.

3. Examples
- Number.isInteger(0) -> true
- Number.isInteger(5.0) -> true
- Number.isInteger(5.000000000000001) -> false
- Number.isInteger('10') -> false
- Number.isInteger(NaN) -> false

SUPPLEMENTARY DETAILS

Implementation notes:
- For custom integer checks for big integers, use BigInt and typeof checks, or compare Math.floor/ceil only for finite numbers.
- To coerce parse result: const n = Number(value); then Number.isInteger(n) after validating !Number.isNaN(n) and Number.isFinite(n).

REFERENCE DETAILS

Specification reference: ECMAScript "# sec-number.isinteger" defines semantics.

Method signature and behavior summary for implementation:
- Input: any JS value
- Steps: if typeof value !== 'number' return false; if !Number.isFinite(value) return false; return Math.floor(value) === value.
