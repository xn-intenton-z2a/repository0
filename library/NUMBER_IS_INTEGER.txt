1. Signature and return value
- Function: Number.isInteger(value)
- Parameters: value — any JavaScript value
- Return: boolean — true if value is a Number value that is an integer (no fractional part) according to ECMAScript numeric representation; false otherwise.

2. Precise behavior and edge cases
- Returns true for numeric integer values (e.g., 0, 1, -100000, 5.0).
- Returns false for NaN, Infinity, -Infinity.
- Returns false for non-number types (strings, booleans, objects, arrays).
- Floating point precision: very small fractional differences may be lost due to IEEE-754 representation; values like 5.0000000000000001 will be represented as 5 and Number.isInteger may return true.
- Large magnitude values near Number.MAX_SAFE_INTEGER may lose precision making Number.isInteger return true for values that appear to have fractional parts when written in decimal.

3. Examples and pitfalls
- Common correct usage to validate integer input:
  if (!Number.isInteger(n)) throw new RangeError('n must be integer');
- Pitfall: using parseInt or Math.floor on non-numeric values may coerce unexpectedly; prefer Number.isInteger to check without coercion.
- Pitfall: trusting decimal literals like 4500000000000000.1; due to precision such values may be treated as integers.

SUPPLEMENTARY DETAILS

Technical specifications and implementation details:
- ECMAScript specification reference: sec-number.isinteger in ECMAScript spec.
- Implementations should not coerce input; Number.isInteger is a predicate that returns false for non-number types.

REFERENCE DETAILS

Exact signature and semantics:
- Number.isInteger(value) -> boolean
- Behavior: return typeof value === 'number' && isFinite(value) && Math.floor(value) === value (conceptual equivalence; spec uses ToNumber and mathematical integer predicate)
