1. Description and use cases
- RangeError is thrown when a value is not within the set or range of allowed values for a function's parameter (e.g., invalid array length, invalid numeric argument to Number.prototype.toFixed, invalid string normalization option).
- Use RangeError to signal invalid numeric ranges and to provide clear, catchable semantic errors.

2. Constructor and instance properties
- Constructor: RangeError(message?) creates a new RangeError object.
- Instance properties inherited from Error: name (initial value 'RangeError'), message, stack (implementation dependent).
- RangeError.prototype.name === 'RangeError'.
- RangeError is serializable and cloneable via structuredClone and transferable via postMessage between Workers.

3. Examples and handling patterns
- Throwing example for numeric range:
  function check(n) { if (!(n >= -500 && n <= 500)) { throw new RangeError('The argument must be between -500 and 500.'); } }
- Throwing example for categorical constraint:
  if (!['apple','banana','carrot'].includes(value)) { throw new RangeError('The argument must be an "apple", "banana", or "carrot".'); }
- Catching pattern:
  try { check(2000); } catch (err) { if (err instanceof RangeError) { /* handle range-specific recovery or message */ } else { throw err; } }
