1. Definition and when to throw
- RangeError is an Error subtype thrown when a value is outside the allowed numeric or enum range for a function/operation.
- Common cases: invalid array length, invalid numeric formatting arguments (toFixed/toExponential errors), invalid normalize argument, and explicit API range checks.

2. Constructor and properties
- Constructor: new RangeError([message]) or RangeError([message]).
- Instance properties: name ("RangeError"), message, stack (inherited from Error), constructor.
- Serializable and transferable via structuredClone/postMessage.

3. Usage examples
- Numeric range validation:
  function check(n){ if (!(n >= min && n <= max)) throw new RangeError("...") }
- Non-numeric enumeration validation:
  if (!allowed.includes(value)) throw new RangeError('argument must be one of ...')

SUPPLEMENTARY DETAILS

Implementation guidance:
- Use RangeError when input is a number but outside allowed range; use TypeError when input has wrong type.
- For APIs accepting size/length, validate with integer checks and range checks before using constructors that can throw implicitly (e.g., new Array(length)).

REFERENCE DETAILS

Exact constructor signature:
- RangeError([message: string]) -> RangeError instance

Best practices:
- Prefer explicit validation and descriptive messages: throw new RangeError("count must be between 1 and 100 inclusive");
- Catch range errors selectively: catch (e) { if (e instanceof RangeError) ... }
