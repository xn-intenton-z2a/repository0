1. Constructor and inheritance
- RangeError is a built-in Error subtype in ECMAScript. Constructor signature: new RangeError([message]) -> RangeError instance.
- Prototype chain: RangeError.prototype instanceof Error.

2. Typical causes
- Numeric value out of allowed range (e.g., invalid array length, invalid repeat count, invalid shift count).
- Invalid arguments to APIs that require specific numeric ranges.

3. Throwing and handling
- Throw new RangeError('message') to signal out-of-range parameter values.
- Catch using try { } catch (e) { if (e instanceof RangeError) { /* handle specific */ } else { throw e } } to only handle range errors.

4. Best practices for diagnostics
- Include parameter name and actual vs allowed range values in the error message to aid debugging.
- For APIs, validate input and throw RangeError before performing operations that assume valid ranges to avoid inconsistent state.
