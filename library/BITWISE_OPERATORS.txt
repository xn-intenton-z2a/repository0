1. JavaScript bitwise operator set
Binary bitwise: &, |, ^  (AND, OR, XOR)
Unary bitwise: ~ (NOT)
Bitwise shifts: << (left), >> (signed right), >>> (unsigned right)
Assignment variants: <<=, >>=, >>>=, &=, ^=, |=

2. Abstract operation semantics (ToInt32 conversion)
Bitwise operators in JavaScript operate on 32-bit signed integers. The runtime performs ToInt32 conversion on numeric operands before the operation. Steps (conceptual):
- Convert operand to Number if not already
- Apply ToInt32: truncate to 32-bit integer per ECMAScript ToInt32 semantics
- Perform bitwise operation at 32-bit level
- Convert result back to Number (IEEE-754 double) when returning
Implication: bitwise operations only cover lower 32 bits; values outside signed 32-bit range are truncated.

3. Operators used for Hamming on numbers
- XOR (^) implements bitwise exclusive or on ToInt32(x) and ToInt32(y). To compute 32-bit Hamming distance: hd32 = popcount((x ^ y) >>> 0) // ensure unsigned view if counting bits as unsigned.
- AND (&) and OR (|) useful for masking and slicing word chunks when constructing popcount tree-based algorithms.
- Complement (~) inverts the 32-bit two's complement representation.

4. Caveats and implementation notes
- Because ToInt32 truncates to 32 bits, for integers larger than 32 bits use multi-word processing (process with 32-bit chunks) or use BigInt bitwise operators where supported (BigInt supports bitwise ops but semantics differ; confirm environment support).
- Using >>> 0 converts a signed 32-bit result into an unsigned 32-bit Number in JavaScript (useful to interpret bit patterns as unsigned when computing popcount).
- When computing popcount in JS for 32-bit values, implement word-level popcount (use Kernighan's method in a loop, or precompute 16-bit table) or use integer bit-twiddling sequences ported from C, ensuring values are masked to 32 bits with >>> 0.

REFERENCE DETAILS (operator signatures and effects)
- a & b: returns ToInt32(a) & ToInt32(b) as Number (signed 32-bit interpreted result converted to Number)
- a | b: returns ToInt32(a) | ToInt32(b)
- a ^ b: returns ToInt32(a) ^ ToInt32(b)
- ~a: returns ~ToInt32(a)
- a << b: left shift of ToInt32(a) by (ToUint32(b) & 0x1F)
- a >> b: arithmetic right shift of ToInt32(a) by (ToUint32(b) & 0x1F)
- a >>> b: logical right shift of ToUint32(a) by (ToUint32(b) & 0x1F)
