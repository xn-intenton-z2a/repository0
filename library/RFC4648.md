Alphabet definitions
- Base64 standard alphabet: "ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789+/"
- Base64url alphabet: "ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789-_" (replaces +/ with -_ for URL safety)
- Base32 and Base16 alphabets are defined by RFC4648 but are outside the immediate mission focus; base64 is used as the baseline for density comparisons.

Encoding block sizes and bit math
- Base64 encodes groups of 3 bytes (24 bits) into 4 output characters (4 * 6 = 24 bits). Each output character represents 6 bits.
- Effective density: 6 bits per output character.
- For arbitrary input length: output length = ceil((input_bytes * 8) / 6).

Padding rules
- Standard base64 uses '=' padding to make the final quantum 4 characters long when input length is not a multiple of 3.
- Padding values:
  - If input length mod 3 == 1 -> output ends with '==' (two padding chars)
  - If input length mod 3 == 2 -> output ends with '=' (one padding char)
- RFC4648 allows implementations to omit padding in contexts where length can be inferred; many web/URL implementations remove padding for compactness.

Base64url variant
- Replace '+' with '-' and '/' with '_'.
- Padding may be omitted; when omitted, decoder must accept both padded and unpadded encodings or reconstruct padding by using input length mod 4.

Enc/Dec algorithm (byte/bit operations)
- Encoding (Uint8Array -> string):
  1. Read input bytes into a bitstream.
  2. Take 6-bit slices from most-significant-bit to least and map each 6-bit value (0..63) to the alphabet character.
  3. For the final incomplete 6-bit slice, add zero bits on the right to form a full 6-bit value; emit output and add '=' padding in standard base64 to reach 4-char block boundary.
- Decoding (string -> Uint8Array):
  1. Map each character to its 6-bit value.
  2. Concatenate 6-bit groups into a bitstream and split into 8-bit bytes.
  3. When padding '=' is present, ignore the corresponding trailing bits.

Supplementary details
- Implementations must treat non-alphabet characters as errors (or ignore them depending on policy); for deterministic round-trip behaviour, reject inputs with invalid characters.
- When supporting base64url, allow both '-'/'_' and '+'/'/' if interoperating, but preserve exact alphabet when producing output.
- Performance: operate on 24-bit words mapped to 4 output chars to minimize bit-level operations in hot paths.

Reference details (precise alphabets and behaviour)
- Standard Base64 alphabet: ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789+/
- Base64url alphabet: ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789-_
- Bits per character: 6
- Block sizes: 3 input bytes -> 4 output characters
- Padding character: '=' (optional in some contexts)
- Exact method signatures (recommended for this library):
  - encode(name: string, input: Uint8Array): string  // name included for API parity with custom encodings
  - decode(name: string, input: string): Uint8Array
  - createFromAlphabet(name: string, alphabet: string): Encoding  // returns object with encode/decode
