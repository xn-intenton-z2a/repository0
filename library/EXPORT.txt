1. Export statement syntaxes
- Export declarations appear in module scope only.
- Syntax variants: export const name = value; export function fn() {}; export class C {}; export { a, b as c };
- Default export: export default expression-or-declaration.
- Re-export: export { name } from 'module'; export * from 'module'; export { default } from 'module';

2. Named exports
- Named exports create live bindings: the exported identifier reflects the original value as it changes in the exporting module.
- Importing modules receive read-only views of these bindings; reassigning imported binding in importer is not allowed.
- Exported identifiers must be declared in the module or exported as aliases from other modules.

3. Default export
- A module may have a single default export. Import syntax: import X from 'module'; resolves to the default export value.
- Default export semantics: the exported value is treated like a named export with the special name "default"; still a live binding if the default is a binding declaration.

4. Re-exports and aggregation
- export { name } from 'module' forwards the exported binding; it does not create a local binding unless explicitly declared.
- export * from 'module' re-exports all named exports except default. Use combined with export { default } from 'module' to include default.
- Export lists can rename: export { a as b } from 'module'.

5. Export semantics and bindings
- Exports are resolved statically at module link time; circular dependencies are permitted but must rely on live binding behaviour to observe updates during execution.
- Syntax errors: export declarations are syntax errors in non-module scripts. Use package.json "type": "module" or .mjs extension to enable ESM.

SUPPLEMENTARY DETAILS
- Tooling: transpilers may transform export syntax; when targeting CommonJS, named exports map to properties on module.exports; default maps to module.exports.default by some transforms unless interop is applied.
- Best practice: prefer named exports for libraries to enable tree-shaking; provide a single default for primary exported value when ergonomics require it.

REFERENCE DETAILS
- Exact constructs:
  - export declaration forms: export declaration (function/class/var/let/const), export list, export default, export from, export * from.
  - Live binding behaviour: importedBinding = referenceToExportedValue (read-only reference in importer).
- Module resolution rules for export from: module specifier follows standard import resolution rules.
