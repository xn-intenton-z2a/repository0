1. Enabling ESM in Node.js
- To treat .js files as ES modules set package.json field: "type": "module".
- Alternatively use .mjs extension to mark files as ESM independent of package.json.

2. import/export support and top-level await
- Native import and export syntax is supported in ESM modules.
- Top-level await is supported in ESM modules: modules may use await at top-level; callers will await module evaluation if they import it.

3. import.meta and file path utilities
- __filename and __dirname are not defined in ESM. Use import.meta.url and utilities:
  - from 'url' import fileURLToPath(url: string) -> string
  - from 'path' import dirname(path: string) -> string
  - Example reconstruction: const __filename = fileURLToPath(import.meta.url); const __dirname = dirname(__filename);

4. JSON and import assertions
- Static import of JSON requires an assertion: import data from './data.json' assert { type: 'json' } ;
- Without assertion, JSON modules are not imported as modules in strict ESM resolution.

5. Module resolution and package exports field
- package.json "exports" field controls which entry points are visible to importers; exports can provide conditional exports keyed by environment conditions (node, import, require, default) and subpath mappings.
- Use "type": "module" to change interpretation of .js; otherwise .cjs indicates CommonJS.

6. Supplementary details (compat, caveats)
- Interop with CommonJS: importing a CommonJS module into ESM yields a default export equal to module.exports; named imports do not map automatically to named exports.
- To import ESM from CommonJS, use dynamic import() which returns a promise resolving to the module namespace.
- Be explicit about file extensions in import specifiers when packages are published (Node requires exact filenames or package exports mapping).

7. Reference details (exact configuration keys and APIs)
- package.json keys: "type": "module" or "type": "commonjs".
- API: fileURLToPath(url: string) -> string (from 'url' module).
- API: path.dirname(p: string) -> string (from 'path' module).
- Import assertion syntax: import X from 'file.json' assert { type: 'json' } ;
