1. Description and When to Throw
- RangeError is a built-in Error subtype thrown when a numeric value is outside an allowed range or an argument is not an allowed value for a function's parameter space.
- Common triggers:
  - Creating an array with illegal length (new Array(-1))
  - Passing out-of-range inputs to Number.prototype.toExponential/toFixed/toPrecision
  - Explicit validation: argument must be between min and max

2. Constructor and Instance Properties
- Constructor: RangeError(message?: string)
- Instance properties inherited from Error: name ("RangeError"), message, stack (engine-dependent).

3. Common Use Cases
- API validation: if (!(n >= min && n <= max)) throw new RangeError("The argument must be between min and max.");
- Non-numeric allowed values for specific APIs: throw RangeError when an option string is not among allowed enumerated values (example: String.prototype.normalize allowed forms).
- Serializable: RangeError is serializable and can be structuredCloned or posted between workers.

4. Example Handling Patterns
- Throwing:
  - if (!Number.isInteger(value) || value < 0 || value > MAX) throw new RangeError('...');
- Catching and filtering:
  - try { fn(); } catch (err) { if (err instanceof RangeError) handleRangeError(err); else throw err; }
