1. Constructor and signature
- Constructor: new RangeError([message: string]) -> RangeError instance
- RangeError is a built-in error type representing an argument outside the allowed range.

2. When RangeError is thrown (common built-ins)
- Examples of host-generated RangeError conditions:
  - Number.prototype.toFixed called with fractionDigits outside allowed range.
  - Array length set to a value outside allowed range.
  - Calling Web APIs or host APIs with numeric arguments exceeding allowed bounds.
- Implementations may throw RangeError when parsing or converting values where the numeric range is violated.

3. Properties and prototype chain

5. Supplementary implementation notes
- Create custom range checks in library code and throw new RangeError('detail') to match host error semantics when argument ranges are invalid.
- Preserve message strings that include exact allowed ranges for diagnostics (e.g., "fractionDigits must be between 0 and 20").
